# AP v2 Launch Gate

## Sprint 30 — Launch Readiness 2

**A system is ready when it can explain its own readiness.**

## Gate Criteria

AP v2 may launch when:

1. The static preflight has no blocking issues.
2. The runtime readiness audit has no launch-blocking issues on major pages.
3. Atlas focus links preserve the correct base path.
4. Relationship grammar reads like meaning, not metadata.
5. Journeys continue intentionally from content pages.
6. Unavailable resources guide the learner rather than creating dead ends.
7. Mobile does not behave like a crushed desktop.
8. The AP visual system reveals relationships without overwhelming the content.
9. The site feels coherent enough to become the public standard.

## Launch Standard

A launch is not a finish line. A launch is a declared standard.

AP v2 launches when the system is ready to teach.
