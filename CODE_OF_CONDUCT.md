# Code of Conduct

## Standard

Participants in KGB Studio spaces are expected to communicate professionally, stay focused on the work, respect intellectual-property boundaries, and treat other participants with dignity.

Constructive disagreement is welcome. Harassment, threats, discrimination, sexualized conduct, deliberate disruption, impersonation, doxxing, spam, or repeated bad-faith behavior is not.

## Project boundaries

Participants must not:

- pressure maintainers to accept work or change project direction;
- misrepresent affiliation with the project;
- publish private security details contrary to `SECURITY.md`;
- submit material they do not have the right to contribute;
- remove ownership notices or encourage unauthorized reuse; or
- use project spaces primarily to promote unrelated products, services, or causes.

## Enforcement

The maintainer may edit or remove content, close or lock discussions, reject contributions, restrict participation, or report conduct to GitHub or appropriate authorities.

Serious or repeated violations may result in permanent exclusion without prior warning.

## Reporting

Report conduct concerns privately to the repository owner through the owner’s GitHub profile. For security vulnerabilities, use the private process in `SECURITY.md`.

This code governs participation only. It does not grant rights to project material.
