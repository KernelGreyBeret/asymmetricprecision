# AP Atlas Cartography

## Sprint 20

### Principle

> A map should reveal structure, not merely prove links exist.

Sprint 19 gave the Atlas its first visible relationship lines. Sprint 20 turns those lines into a more intentional cartographic surface.

## What changed

- Added map density controls: Quiet, Balanced, Full System.
- Added relationship label controls: Active only, Visible edges, Hide labels.
- Added curved relationship paths instead of simple straight lines.
- Added relationship groups: structure, learning, application, evidence, visualization, relationship.
- Added a relationship legend directly on the map.
- Added domain hulls to make clusters feel like conceptual regions.
- Added map metrics so the visitor knows what projection they are seeing.
- Strengthened Atlas URL state for projection, density, labels, and focus.

## Why this matters

The Atlas is not just navigation. It is the visual proof that AP is a connected body of work. People should be able to see structure, density, paths, and conceptual neighborhoods without guessing how the system fits together.

## Design constraint

More lines are not automatically better. The Atlas should allow density without forcing clutter. Quiet, Balanced, and Full System modes exist because different questions require different projections.
