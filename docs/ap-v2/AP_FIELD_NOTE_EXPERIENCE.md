# AP Field Note Experience

## Principle

A field note is not a post. A field note is an observation.

## Purpose

Field Notes capture grounded signal from lived systems. They are intentionally shorter than essays and less formal than frameworks, but they still connect into the AP knowledge system.

## Field Note Structure

Each Field Note should identify:

- the observed signal
- the possible system meaning
- the domains it touches
- the AP concepts it connects to
- where the reader can continue exploring

## Design Rule

Do not write Field Notes to publish reactions.

Write them to preserve observations before the system explains them away.

## AP Integration

Field Notes connect to:

- Atlas nodes
- essays as lenses
- frameworks as operating models
- books as curriculum paths
- learning checkpoints
- journeys
- laboratories

## Success Test

Did the Field Note help someone notice a system behavior they might otherwise have missed?
