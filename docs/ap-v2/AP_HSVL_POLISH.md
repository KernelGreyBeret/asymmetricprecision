# AP HSVL Polish

## Sprint 13 Principle

**A visual system should reveal relationships, not decorate content.**

Hidden Systems Visual Language is not an aesthetic skin. It is a method for making invisible structure easier to perceive.

## What HSVL Should Reveal

- Relationships
- Constraints
- Evidence
- Unknowns
- Feedback loops
- State changes
- Connection paths
- System boundaries

## What HSVL Should Avoid

- Decoration without meaning
- Excessive animation
- Sci-fi styling that reduces trust
- Visual noise
- Components that look interesting but explain nothing

## Core Patterns

### Telemetry Labels

Small status indicators that expose state, domain, confidence, or relationship.

### Connection Bands

Vertical or horizontal relationship paths showing how ideas connect.

### System Map Surfaces

Panels that frame grouped content as interacting system elements.

### Evidence Surfaces

Callouts that separate observation from assumption.

### System Annotations

Brief interpretive notes that name the hidden structure shaping visible behavior.

## Design Rule

If the visual treatment does not help someone understand the system more clearly, remove it.
