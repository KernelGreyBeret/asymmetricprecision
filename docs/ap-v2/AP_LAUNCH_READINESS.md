# AP v2 Launch Readiness Checklist

## Identity

- [ ] Homepage communicates AP in five seconds.
- [ ] Oath appears early enough to set the tone.
- [ ] Mission language is consistent across Home, About, Philosophy, and Contact.
- [ ] Cybersecurity is framed as an application, not the whole identity.
- [ ] AP vocabulary is used consistently.

## Navigation

- [ ] Top nav works on desktop.
- [ ] Top nav works on mobile.
- [ ] Active states are visible.
- [ ] Legacy `projects/` path redirects to `laboratories/`.
- [ ] No primary page feels orphaned.

## Atlas

- [ ] Atlas loads data.
- [ ] Search works.
- [ ] Type filters work.
- [ ] Domain filters work.
- [ ] Node detail panels work.
- [ ] Connection neighborhoods render clearly.

## Content Graph

- [ ] Books index loads.
- [ ] Essays index loads.
- [ ] Frameworks index loads.
- [ ] Field Notes index loads.
- [ ] Laboratories index loads.
- [ ] Metadata surfaces are visible and helpful.
- [ ] “Where this connects” surfaces appear where intended.

## Learning System

- [ ] Reflection checkpoint works locally.
- [ ] Observation capture works locally.
- [ ] Reflection log works locally.
- [ ] Export works.
- [ ] Clear works.
- [ ] No interaction requires a backend.

## Accessibility

- [ ] Keyboard navigation works.
- [ ] Focus states are visible.
- [ ] Skip link works.
- [ ] Reduced motion preference is respected.
- [ ] Touch targets are usable on mobile.
- [ ] Text remains readable on small screens.

## Performance

- [ ] No console errors on major pages.
- [ ] No missing CSS/JS files.
- [ ] No missing data files.
- [ ] No broken primary internal links.
- [ ] Pages remain usable if optional JS fails.

## Launch Decision

- [ ] AP v2 feels like one coherent system.
- [ ] The preview is stable enough to replace the current live site.
- [ ] Remaining issues are known and non-blocking.
