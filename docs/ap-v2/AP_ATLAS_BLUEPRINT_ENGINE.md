# AP Atlas Blueprint Engine

## Sprint 19 Principle

A map is not navigation. A map is a model of relationship.

## What Changed

Sprint 19 makes the Atlas behave less like a filterable archive and more like a system blueprint.

It adds four projection modes:

- **System Map** — the visible AP knowledge system arranged by domain clusters.
- **Focused Neighborhood** — the selected node and the things directly connected to it.
- **Domain Cluster** — the selected node's domain as a local operating environment.
- **Journey Path** — a guided path projected as a sequence through the system.

## Why It Matters

AP should not merely say that relationships matter.

It should show them.

The Atlas now treats filters as projections of one underlying map rather than separate views that invent their own structure.

## Design Rule

The Atlas should answer three questions:

1. What am I looking at?
2. What is it connected to?
3. Why does that connection matter?

If the map does not help answer those questions, it is decoration and should be removed or redesigned.
