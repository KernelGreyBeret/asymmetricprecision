# AP v2 Polish Sprint Report

## Principle

Coherence is not decoration. Coherence is how a system makes itself easier to understand.

## What changed

- Added the `ap-polish.css` layer for active navigation, focus states, mobile navigation clarity, and reduced-motion support.
- Added the `ap-polish.js` layer for small system-level behaviors that reduce friction without adding dependencies.
- Added token aliases so all system layers share the same vocabulary (`--ap-accent`, `--ap-space-*`, `--ap-radius-*`, `--ap-shadow-soft`).
- Updated the homepage Atlas language now that the Atlas exists.
- Added loading-state conventions for surfaces that hydrate through local JavaScript.

## What this sprint intentionally did not do

- It did not add new major site sections.
- It did not introduce external services.
- It did not add engagement mechanics.
- It did not change AP's visual identity.

## Test

A polish change ships only when it reduces uncertainty, clarifies the system, or makes an existing relationship easier to perceive.
