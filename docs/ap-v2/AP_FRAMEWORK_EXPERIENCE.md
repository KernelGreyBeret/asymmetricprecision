# AP Framework Experience

## Principle

**A framework is not a document. A framework is an operating model.**

A framework should not merely explain an idea. It should help someone apply an idea to a real system.

## Required Framework Elements

Every AP framework should eventually include:

1. **Problem** — what confusion, risk, failure pattern, or decision problem does it address?
2. **Model** — what is the structure of the idea?
3. **Application** — how does someone use it?
4. **Evidence Questions** — what would prove the model is working?
5. **Connections** — where does it sit in the Atlas?
6. **Reflection** — did it improve understanding?

## Design Intent

Frameworks are bridges between thinking and action.

They should help leaders, engineers, architects, analysts, and builders move from vague concern to structured understanding.

## AP Rule

If a framework cannot be applied, it is not finished.
