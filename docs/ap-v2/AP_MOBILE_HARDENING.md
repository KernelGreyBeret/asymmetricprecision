# AP Mobile Hardening

## Principle

**Small screens should reduce uncertainty, not hide the system.**

Mobile AP should not feel like a compressed desktop. It should feel like a clearer path through the same philosophy.

## Sprint 14 Adds

- Minimum touch-target sizing
- Responsive grid collapse rules
- Atlas/detail layout fallbacks
- Better mobile nav state behavior
- Readability constraints for long copy
- Overflow protection for visual-language surfaces

## Rule

Mobile is not a downgrade. Mobile is a constrained environment that forces clarity.
