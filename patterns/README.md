# AP Patterns

Reusable surfaces for Asymmetric Precision v2.

## Principle

Patterns are not snippets. Patterns are reusable ways of teaching the system.

## Current Patterns

- `atlas-explorer.html` — turns the Atlas into an exploration surface.
- `atlas-node.html` — describes an Atlas node.
- `connection-panel.html` — shows relationships as first-class citizens.
- `content-card.html` — turns content into a learning surface.
- `metadata-surface.html` — exposes thinking metadata.
- `learning-checkpoint.html` — asks whether understanding improved.
- `observation-capture.html` — captures unresolved questions locally.
- `reflection-log.html` — lets the visitor review their local learning trail.

## Sprint 6 Rule

A pattern only belongs here if reuse increases clarity.
