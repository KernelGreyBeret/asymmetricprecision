# AP v2 Sprint 22 — Contextual Surfaces

Principle: **Context should travel with the learner.**

Sprint 22 makes AP pages behave less like isolated destinations and more like situated surfaces inside the larger system.

## Adds

- System Context strip for content pages
- Read-along contextual panels for essays/books/content detail pages
- Focused Atlas links generated from the current page
- Contextual continuation surfaces
- Journey-aware continuation when a page is part of a path
- Boundary-safe contextual card styling

## Files

- `assets/system/ap-contextual-surfaces.js`
- `assets/system/ap-contextual-surfaces.css`
- `assets/system/ap-system.js`
- `assets/system/ap-system.css`
- `docs/ap-v2/AP_CONTEXTUAL_SURFACES.md`

## Why this matters

AP should not make the learner reconstruct context by hand. If an essay connects to principles, frameworks, books, field notes, laboratories, or journeys, the page should show that relationship directly and offer the next useful move.
